# 거상 To-Do List - Handoff 문서

## 프로젝트 개요
거상(온라인 게임) 플레이어를 위한 일일/주간 퀘스트 및 던전 공고 관리 To-Do List 웹 앱.
순수 HTML/CSS/JS(바닐라)로 구현, 외부 라이브러리 없음.

- **배포 URL**: https://helperjby.github.io/gersang-to-do-list/
- **저장소**: https://github.com/helperjby/gersang-to-do-list
- **배포 방식**: GitHub Pages (main 브랜치, / 경로)

---

## 파일 구조

## 주요 기능

### 1. 카테고리 (4개)
| 카테고리 | 키 | 리셋 |
|---------|-----|------|
| 해야 할 일 | `todo` | 없음 |
| 일일 퀘스트 | `daily` | 매일 06:00 |
| 주간 퀘스트 | `weekly` | 매주 일요일 06:00 |
| 이벤트 | `event` | 없음 |

### 2. 퀘스트 관리
- **추가**: 입력란 + 버튼 (Enter키 지원)
- **완료 체크**: 체크박스 토글, 완료 항목 자동 하단 정렬
- **삭제**: − 버튼
- **인라인 편집**: 퀘스트 이름 더블클릭 → input 전환 (Enter/blur 저장, Escape 취소)
- **순서 변경**: ⇅ 버튼 토글 → 드래그 핸들(☰) 표시 → 마우스 이벤트 기반 드래그앤드롭으로 순서 변경
- **진행률**: 카테고리별 완료/전체 카운트 + 프로그레스 바

### 3. 던전 공고(반복) 패널
- 헤더 "⚔ 던전 공고" 버튼으로 토글 (show/hide 상태 localStorage 저장)
- 2컬럼 레이아웃: 좌 480px(투두) / 우 380px(던전)
- **고정 20개 던전 목록**: 고수동굴, 대관령, 한라산, 월기봉, 거제해저동굴, 무령왕릉, 천년호, 이시즈치산, 하치만타이온천, 대설산, 일본해저동굴, 귀곡성, 대둔산, 대만해저동굴, 해적동굴, 해적동굴2층(200렙 이하), 유명계, 륭산, 샤오링의후원, 챠우신전
- **삭제**: ✕ 버튼으로 개별 던전 제거 (hidden 처리, localStorage 유지)
- **리셋**: ↻ 리셋 버튼 → 전체 목록 복원 + 체크 초기화 (디폴트 복원)
- **일일 리셋(06:00)**: 체크만 초기화, 삭제된 목록은 유지
- 진행률 카운트 + 프로그레스 바

### 4. 자동 리셋
- **일일 리셋**: 매일 06:00 기준, `daily` 퀘스트 + `dungeon` 체크 초기화
- **주간 리셋**: 매주 일요일 06:00 기준, `weekly` 퀘스트 초기화
- 헤더에 마지막 리셋 시각 표시

## 데이터 구조 (localStorage)

키: `gersang-todo`

```json
{
  "todo": [{ "id": "xxx", "name": "할일명", "done": false }],
  "daily": [...],
  "weekly": [...],
  "event": [...],
  "dungeon": [{ "name": "고수동굴", "done": false, "hidden": false }, ...],
  "showDungeon": true,
  "lastDailyReset": "2026-03-28T21:00:00.000Z",
  "lastWeeklyReset": "2026-03-21T21:00:00.000Z"
}
```

- 퀘스트: `id`(자동 생성), `name`, `done`
- 던전: `name`(고정), `done`, `hidden`
- 마이그레이션: 기존 `dailyRepeat` → `todo` 자동 이관

## 알려진 사항 / 참고
- PC 전용 레이아웃 (모바일 뷰 미고려)
- 외부 라이브러리 없음 (순수 바닐라 JS)
- 다크 테마 고정 (#1a1a2e 배경, #e8b04a 골드 포인트)
- GitHub Pages 배포 시 CDN 캐시로 반영까지 1~2분 소요될 수 있음
- CSS/JS 파일 변경 시 index.html의 쿼리 스트링 버전(`?v=N`)을 올려야 캐시 갱신됨
